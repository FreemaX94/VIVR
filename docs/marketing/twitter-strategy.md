# VIVR - Strategie Marketing Twitter/X

## Vue d'ensemble de la marque

**VIVR** est une boutique e-commerce francaise de decoration interieure specialisee dans un style minimaliste et elegant. Le slogan "Vivez votre interieur" capture parfaitement l'essence de la marque : transformer les espaces de vie en lieux de bien-etre.

### Valeurs de marque
- Minimalisme elegant
- Qualite artisanale
- Design scandinave et contemporain
- Accessibilite du beau design

### Categories de produits
- Salon (lampes, vases, miroirs)
- Chambre (coussins, textiles)
- Cuisine
- Bureau
- Salle de bain
- Exterieur

## 2. Tactiques d'engagement

### 2.1 Construction de communaute

#### Strategie de reponse
- Repondre a TOUTES les mentions dans l'heure
- Utiliser le prenom quand disponible
- Poser une question de suivi pour maintenir la conversation
- Remercier sincerement pour les compliments

**Templates de reponse** :

```
Reponse positive :
"Merci beaucoup [Prenom] ! On adore voir nos pieces dans de si beaux interieurs. Vous avez un talent pour la mise en scene !"

Reponse a une question :
"Excellente question ! [Reponse]. N'hesitez pas si vous avez besoin d'autres conseils pour votre projet. On est la !"

Reponse a un probleme :
"Oh non, on est vraiment desole pour ce desagrement. On vous envoie un DM immediatement pour arranger ca."
```

#### Strategie proactive
- Rechercher et commenter les posts #decoration #interieurdesign
- Feliciter les nouveaux proprietaires (recherche "nouvel appartement" "nouvelle maison")
- Participer aux conversations sur le design
- Soutenir les createurs de contenu deco

### 2.3 Engagement influenceurs

#### Types d'influenceurs cibles

**Macro-influenceurs (50K-500K followers)**
- Architectes d'interieur reconnus
- Blogueuses lifestyle premium
- Createurs de contenu home

**Micro-influenceurs (5K-50K followers)**
- Passionnes de deco avec communaute engagee
- Jeunes architectes d'interieur
- Comptes lifestyle regionaux

**Nano-influenceurs (<5K followers)**
- Clients satisfaits avec beaux interieurs
- Fans de design actifs sur Twitter

#### Strategie d'approche

**Phase 1 : Observation (2 semaines)**
- Suivre leurs contenus
- Liker et commenter authentiquement
- Identifier leurs preferences et style

**Phase 2 : Interaction (2 semaines)**
- Partager leurs contenus avec valeur ajoutee
- Repondre a leurs questions
- Les mentionner dans des contextes pertinents

**Phase 3 : Collaboration (apres 1 mois)**
- Proposition de produit a tester
- Invitation a des evenements
- Collaboration de contenu

**Template de message DM** :
```
Bonjour [Prenom],

Chez VIVR, on suit votre travail depuis un moment et on adore votre approche du design [mention specifique].

On aimerait vous faire decouvrir notre collection [categorie].
Pas d'obligation, juste l'envie de vous faire plaisir et de connaitre votre avis.

Interessee ?

L'equipe VIVR
```

```tsx
  limit?: number
```

```tsx
// components/social/TweetEmbed.tsx
'use client'

import { useEffect, useRef } from 'react'

interface TweetEmbedProps {
  tweetId: string
  className?: string
}

export function TweetEmbed({ tweetId, className }: TweetEmbedProps) {
  const containerRef = useRef<HTMLDivElement>(null)

  useEffect(() => {
    // Load Twitter widgets script
    const script = document.createElement('script')
    script.src = 'https://platform.twitter.com/widgets.js'
    script.async = true
    document.body.appendChild(script)

    script.onload = () => {
      if (window.twttr && containerRef.current) {
        window.twttr.widgets.createTweet(tweetId, containerRef.current, {
          theme: 'light',
          lang: 'fr',
          dnt: true,
        })
      }
    }

    return () => {
      document.body.removeChild(script)
    }
  }, [tweetId])

  return <div ref={containerRef} className={className} />
}

// Utilisation sur une page produit
```

### 3.5 API Route pour le feed Twitter

```typescript
// app/api/social/twitter-feed/route.ts
import { NextRequest, NextResponse } from 'next/server'

// Cache the response for 15 minutes
export const revalidate = 900

export async function GET(request: NextRequest) {
  const { searchParams } = new URL(request.url)
  const hashtag = searchParams.get('hashtag') || 'VIVRchezvous'
  const limit = parseInt(searchParams.get('limit') || '6')

  try {
    // Note: This requires Twitter API v2 Bearer Token
    const response = await fetch(
      `https://api.twitter.com/2/tweets/search/recent?query=%23${hashtag}&max_results=${limit}&tweet.fields=created_at,public_metrics&expansions=author_id&user.fields=name,username,profile_image_url`,
      {
        headers: {
          'Authorization': `Bearer ${process.env.TWITTER_BEARER_TOKEN}`,
        },
      }
    )

    if (!response.ok) {
      throw new Error('Twitter API error')
    }

    const data = await response.json()

    // Transform data to our format
    const tweets = data.data?.map((tweet: any) => {
      const author = data.includes.users.find(
        (user: any) => user.id === tweet.author_id
      )

      return {
        id: tweet.id,
        text: tweet.text,
        author: {
          name: author?.name || '',
          username: author?.username || '',
          avatar: author?.profile_image_url || '',
        },
        metrics: {
          likes: tweet.public_metrics?.like_count || 0,
          retweets: tweet.public_metrics?.retweet_count || 0,
          replies: tweet.public_metrics?.reply_count || 0,
        },
        createdAt: tweet.created_at,
      }
    }) || []

    return NextResponse.json({ tweets })
  } catch (error) {
    console.error('Twitter feed error:', error)

    // Return cached/fallback data
    return NextResponse.json({
      tweets: [],
      error: 'Unable to fetch tweets',
    })
  }
}
```
